# LP Manager Follow-up Tasks (generated 2026-04-07)

Generated by Architect/Analyst agent after LP Inspector CLI (original 12-task plan) was merged to master.

## Summary

The LP Inspector CLI ships and works: 24 unit tests pass, `cargo build` and `cargo clippy -- -D warnings` are clean. However the shipped code has several CLAUDE.md violations papered over with `#[allow(dead_code)]` and inline magic numbers (hardcoded `1e9` / `1e6` decimals, hardcoded "A"/"B" symbols, hardcoded fee scaling, no entry-price tracking so IL is always 0). The Raydium protocol path is half-built (parses, but no analytics/display integration). There is no `README.md`, no `tests/` integration directory, no proptest property-based math suite (CLAUDE.md explicitly requires this), and no validation against the Orca Whirlpool JS SDK reference. The data/strategy/execution/storage layers from CLAUDE.md's vision are entirely unstarted.

## Findings by Category

**(A) Bugs / lints suppressed in shipped LP Inspector**

1. `src/main.rs:121-127` — P&L hardcodes `1e9`/`1e6` decimal scaling, treats token B as USD. Wrong for any non-SOL/USDC pool. Symbols hardcoded to "A"/"B".
2. `src/main.rs:124` — `compute_il(0.0, ...)` always passed entry price 0, so IL is *always* 0. Output is fake.
3. `src/main.rs:138` — fee rate labeling wrong (Orca stores hundredths-of-a-bip, not bps).
4. `src/main.rs:79-82, 169-172, 238-241, 259-262` — `to_f64_price` closure duplicated 4 times; lossy `as f64` on u128 > 2^53.
5. Watch command — no reconnect, no ping/pong, any bad message panics via `?`. CLAUDE.md mandates reconnect.
6. Raydium branch — only prints raw fields; does not call `analytics::*` or `display::table`.
7. All protocol structs use `#[allow(dead_code)]` on every field. CLAUDE.md forbids hiding warnings.
8. Never verifies account owner matches program ID before deserializing. CLAUDE.md: "Always verify program owner before deserializing."
9. `src/protocols/raydium.rs` — field order not verified against program source.
10. `build_distribution` in `depth.rs` is dead; Depth command falls back to misleading pool-level estimate.
11. `greeks.rs:29-30` — uses `1.0001f64.powi(tick)` instead of shared `tick_index_to_sqrt_price`.
12. Impact command duplicates math instead of calling `analytics::depth::estimate_impact`.
13. `rpc.rs` — no timeout, no retry, no commitment level configuration.
