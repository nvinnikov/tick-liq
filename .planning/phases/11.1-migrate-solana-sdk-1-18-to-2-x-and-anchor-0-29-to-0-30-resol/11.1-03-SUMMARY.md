## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 — Blocker] `cargo fix --edition` could not auto-apply edition-2024 fixes; 7 ref-binding sites fixed manually**

- **Found during:** Task 2 first `cargo fix --edition` after switching to edition 2024.
- **Issue:** Edition 2024 tightens match ergonomics: `if let Some(ref x) = &opt` (where `opt` is `Option<T>`) now fails with `error: cannot explicitly borrow within an implicitly-borrowing pattern` because the non-reference pattern `Some(ref x)` implicitly borrows the contents of `&opt` — making the `ref` modifier both redundant and explicitly disallowed. `cargo fix --edition` bailed out at 4 compile errors before reaching all 7 sites and thus could not auto-apply its own suggestions.
- **Fix:** Manually applied the idiomatic edition-2024 equivalent at each site:
  - Two patterns used: (a) drop `ref` when matching `&Option<T>` directly (implicit auto-ref handles it: `if let Some(x) = &opt`); (b) use `.as_ref()` when the scrutinee is an owned `Option<T>` that we still want to borrow (`if let Some(x) = opt.as_ref()`).
  - Files touched: `src/main.rs` lines 760, 929, 978, 1081, 1115, 1178, 1210 (7 sites).
  - Semantic equivalence: all 7 sites previously obtained `&T` from `&Option<T>`; the new forms do the same. No value ownership changes; no signature changes; no business logic touched.
- **Verification:** `cargo build --all-targets && cargo test --all && cargo clippy --all-targets -- -D warnings` all exit 0 after fixes. 336 passed / 0 failed (same count as Plan 02 on `d69a287`).
- **Commit:** `49f9f6f`.

**2. [Rule 3 — Baseline sanity noise on 2021 edition] cargo fix on edition 2021 produced 5 rewrites in src/main.rs; reverted per plan option (A)**

- **Found during:** Task 1 dry-run.
- **Issue:** Running `cargo fix --edition --all-targets --allow-dirty` while still on edition 2021 caused the 2021-edition fixer to rewrite 5 sites in src/main.rs — including one that transformed `if let Err(e) = ... { warn } else { info }` into `match ... { Err => { warn } _ => { info } }`, which is valid but less idiomatic than the original. Per RESEARCH §"Edition 2024 / Resolver 3 Pitfalls" and plan Task 1 guidance, cargo fix on the latest pre-switch edition should be a no-op; the rewrites were a 2021-edition housekeeping artefact, not a 2024-edition migration requirement.
- **Fix:** Reverted via `git checkout -- src/main.rs` (plan option A), leaving Task 2 to re-run cargo fix after edition=2024. The post-edition-2024 fix path produces cleaner, more targeted output (ref-binding removal only).
- **Files modified:** none (revert).
- **Commit:** none (Task 1 acceptance criteria forbid new commits).
