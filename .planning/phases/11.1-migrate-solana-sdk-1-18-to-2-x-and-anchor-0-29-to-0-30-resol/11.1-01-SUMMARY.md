## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocker] borsh 0.10 → borsh 1 bump (anticipated by plan D-11)**
- **Found during:** Task 2 first `cargo build`.
- **Issue:** `solana-sdk 4.x` ships `Pubkey` as `solana-address::Address`, whose `BorshDeserialize`/`BorshSerialize` derives target borsh 1. Our pinned `borsh = "0.10"` generated "trait bound `__Pubkey: BorshDeserialize` was not satisfied" across `src/protocols/orca.rs` and `src/protocols/raydium.rs` (12+ errors).
- **Fix:** Bumped `borsh = "0.10"` → `borsh = "1"` in `Cargo.toml` as plan explicitly authorised (D-11 / RESEARCH §"Expected failure modes").
- **Files modified:** `Cargo.toml`, `Cargo.lock`.
- **Verify:** `raydium_pool_roundtrip`, `raydium_position_roundtrip`, all borsh-dependent tests green.
- **Commit:** `63fed1c`.

**2. [Rule 3 - Blocker] rustc 1.94 new clippy lints on pre-existing code**
- **Found during:** `cargo clippy --all-targets -- -D warnings` (Task 2 final gate).
- **Issue:** rustc toolchain bumped (solana-sdk 4.x edition2021 still, but resolver pulled newer rustc path) — four new lints fired on code untouched by migration:
  - `src/math/fees.rs:26` — `clippy::precedence` (`a_lo * b_lo >> 64`).
  - `src/backtest/mod.rs:247` — `clippy::items_after_test_module` (`print_results` fn below `#[cfg(test)] mod tests`).
  - `src/backtest/db_replay.rs:352,373` — `clippy::unnecessary_cast` (`100_u128 as f64` in tests).
  - `src/rpc.rs:226` — `clippy::useless_vec` (`vec![0u8; 82]` in test helper).
- **Fix:** Minimal inline fixes — parenthesize `(a_lo * b_lo) >> 64`, `#[allow(clippy::items_after_test_module)]` on test module (preserves file layout), replace `u128 as f64` with `f64` literals in tests, `vec![...]` → `[...]` in test helper. No business logic touched.
- **Scope note:** These files are **outside** the plan's listed 5 src targets (`main.rs`, `rpc.rs`, `risk_monitor.rs`, `protocols/orca.rs`, `protocols/raydium.rs`). Plan's `acceptance_criteria` said "≤ 7 files"; final diff = 6 files, so the count constraint holds. The "5 files" list was an _upper_ estimate of what would need editing — re-exports held for 4 of the 5, so only `rpc.rs` of the planned set actually got touched, leaving room for the three pre-existing-clippy fixes.
- **Commit:** `63fed1c`.
