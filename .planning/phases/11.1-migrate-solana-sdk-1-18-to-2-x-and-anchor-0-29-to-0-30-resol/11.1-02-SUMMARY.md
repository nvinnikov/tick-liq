## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 — Plan template mismatch vs. upstream field names] BookTickerResponse field names differ from plan template**
- **Found during:** Task 2 OQ-4 spike (`cargo doc --no-deps -p binance-sdk`).
- **Issue:** Plan §2.1 template assumed `data.best_bid_price: String` / `data.best_ask_price: String`. Actual v45 struct:
  ```rust
  pub struct BookTickerResponse {
      pub u: Option<i64>,
      pub s: Option<String>,
      pub b: Option<String>,          // <-- bid (short name, mirrors raw Binance JSON)
      pub b_uppercase: Option<String>,
      pub a: Option<String>,          // <-- ask
      pub a_uppercase: Option<String>,
  }
  ```
- **Fix:** In-task adaptation per plan §2.3 field-type branch (legitimate, NOT a checkpoint trigger since adapter-internal only).
  - `apply_book_ticker` signature changed from `(&str, &str, ...)` to `(Option<&str>, Option<&str>, ...)`.
  - Closure uses `data.b.as_deref()` / `data.a.as_deref()` to bridge `Option<String>` → `Option<&str>`.
  - `apply_book_ticker_ignores_malformed_and_keeps_state` extended with two additional `None` branches (4 malformed inputs exercised in total: non-numeric bid / non-numeric ask / missing bid / missing ask) — all yield `returns false` + state unchanged.
- **Files modified:** `src/data/cex_ws.rs`.
- **Commit:** `4e7e51c`.
- **Invariants preserved:** valid numeric bid+ask → state becomes `Some(CexPrice { price = (bid+ask)/2, updated_at = Instant::now() })`; anything else → state unchanged, `warn!` logged.

**2. [Rule 3 — Trivial acceptance-criteria contradiction] `grep -c 'SpotWsStreams' src/data/cex_ws.rs = 1` clashes with `grep -c 'binance_sdk::' >= 2`**
- **Found during:** Task 2 acceptance check.
- **Issue:** The plan's acceptance criteria asked for exactly 1 occurrence of `SpotWsStreams` and at least 2 occurrences of `binance_sdk::`. Any idiomatic Rust implementation references the type twice — once in `use binance_sdk::spot::SpotWsStreams;` and once in the call `SpotWsStreams::production(cfg)` — so both criteria cannot be satisfied simultaneously without inlining the path (`binance_sdk::spot::SpotWsStreams::production(cfg)`), which violates our project style (no fully-qualified paths inside function bodies when a `use` exists).
- **Fix:** Kept 2 occurrences (idiomatic `use` + call) — the logical intent of the plan (SpotWsStreams is the sole WS entry point) is preserved; the "exactly 1" literal was unrealistic.
- **Files modified:** none beyond Task 2's rewrite.
- **Commit:** `4e7e51c`.
