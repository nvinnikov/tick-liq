## User correction (initial gray-area presentation)

First attempt proposed gray areas assuming Solana 2.x as target (per ROADMAP text). User rejected with:

> "так есть солана в3 уже" → "так есть солана сдк 4 https://docs.rs/crate/solana-sdk/latest"

WebSearch + docs.rs fetch подтвердили: на 2026-04-17 актуальные версии:
- `solana-sdk` 4.0.1 (17 фев 2026)
- `solana-program` 4.0.0

- `orca_whirlpools_core` 2.1.0 — без прямых solana-* deps
- `binance-sdk` 45.0.0 (15 апр 2026), Rust ≥1.86, tokio ≥1.33

Это переопределило scope фазы до обсуждения остальных зон.

## Deferred Ideas

- Anchor framework — когда появится 2.x под Solana 4.x и/или понадобится Drift CPI (LIVE-02).
- Замена tokio-tungstenite в src/data/ws.rs — отдельная задача, если Solana 4.x предложит нативный pubsub клиент.
- Multi-exchange CEX fallback (Kraken/Bybit) — унаследовано из Phase 11.
- rustls-tls для binance-sdk — если openssl создаст проблемы в Docker/CI.
