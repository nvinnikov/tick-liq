### Обработка ожидаемых issues

**Если `cargo fix` вносит правки в src/**:**
- Ревью каждого изменения — должно быть ТОЛЬКО auto-qualification `Future::poll` / `IntoFuture::into_future` / небольшие idiom rewrites.
- НЕ принимать правки, которые меняют бизнес-логику (сигнатуры функций, semantic closures, обработку ошибок).
- Если cargo fix сломал что-то не-тривиальное — поднять CHECKPOINT.
- **Все принятые правки остаются в working tree для слияния в единственный commit Task 3 (D-11 атомарность).**

**Если `cargo build` выдаёт "error[E0658]: gen is a keyword in the 2024 edition":**
- Grep `grep -rnE '\bgen\b' src/`. RESEARCH утверждает — таких идентификаторов нет. Если всплыл — переименовать на `generator` / `g` / etc.

**Если `cargo clippy -- -D warnings` выдаёт новые lints:**
- Для `clippy::missing_docs_in_private_items`, `clippy::must_use_candidate` — `#[allow(...)]` на crate-level в `src/lib.rs` НЕ добавлять без обсуждения. Вместо этого — починить конкретную функцию или поднять CHECKPOINT.
- Для `clippy::uninlined_format_args` — принять автофикс.
- Для других edition-specific lints — применить минимальный фикс, который не меняет поведение.

**Если resolver 3 даунгрейдит какой-то transitive crate:**
- Посмотреть `cargo update` diff. Если важный крейт (tokio, serde, sqlx) даунгрейднут — поднять CHECKPOINT с выводом `cargo tree --depth 1`.
- Если мелкий тул-крейт (memchr, aho-corasick) даунгрейднут — ОК.
