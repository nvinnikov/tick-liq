## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 — Blocker] Root `audit.toml` was never loaded by cargo-audit 0.22; moved to `.cargo/audit.toml`**

- **Found during:** Task 2 first `cargo audit` run. Adding the 4 new ignore entries to the root `audit.toml` had no effect — advisories continued to surface. Empirical verification by copying the file to `.cargo/audit.toml` (the path encoded in the cargo-audit 0.22.1 binary) produced a clean `cargo audit = 0` exit.
- **Issue:** cargo-audit 0.22 searches only `.cargo/audit.toml` relative to the workspace root. The project's prior `audit.toml` at the repo root was vestigial — it documented intent but did not actually suppress advisories at audit time. This means Plans 01-03's exit gates (none of which ran `cargo audit`) happened to pass on accident; had any of them run audit against the old file, it would have flagged the 4 historical CVEs that the file was supposed to hide.
- **Fix:** `git mv audit.toml .cargo/audit.toml` within the single Plan 04 commit. Updated content lives only in the new location.
- **Files modified:** `audit.toml` deleted, `.cargo/audit.toml` created.
- **Commit:** `ea4bfaf`.
- **Scope impact:** Plan listed `audit.toml` as the single affected file. The new path is semantically the same file (same role, same format) at its tool-canonical location. Final diff includes both the add and the delete; `cargo audit` now actually reads the ignore list.

**2. [Rule 3 — Blocker resolution] 1 medium-severity advisory (RUSTSEC-2026-0009, time 0.3.45 DoS) has no in-MSRV fixup**

- **Found during:** Task 2 `cargo audit` after historical ignores removed.
- **Issue:** `time 0.3.45` — CVSS 6.8 DoS via stack exhaustion in x509 parsing path; upstream fix is `time >= 0.3.47`, but that version requires Rust 1.88.0. Plan 03 explicitly pinned MSRV = 1.86 (D-04, mandated by binance-sdk v45). `cargo update -p time --precise 0.3.47` dry-run confirmed the transitive set (`time 0.3.47`, `time-core 0.1.8`, `time-macros 0.2.27`, `num-conv 0.2.1`) all require Rust 1.88.0. Local fixup without violating the MSRV contract is not available.
- **Checkpoint classification:** Scenario C/D would apply for high/critical severity; medium (6.8) with no in-MSRV fix falls into extended Scenario B handling per user's autonomous checkpoint rules. Attack surface (malformed x509 cert during TLS handshake to operator-configured Solana RPC endpoints) is low exploitability in tick-liq's trust model.
- **Fix:** Added to `.cargo/audit.toml` with full D-13/D-14 justification. Expiry condition: reassess when MSRV can be bumped past 1.88 or when Solana 4.x GA relaxes the x509-parser transitive constraint.
- **Commit:** `ea4bfaf`.

**3. [Rule 3 — Info-severity] 3 unmaintained transitive crates (bincode 1.3.3, proc-macro-error 1.0.4, rustls-pemfile 1.0.4)**

- **Found during:** Task 2 `cargo audit` output.
- **Issue:**
  - `bincode 1.3.3` (RUSTSEC-2025-0141): pulled by 20+ `solana-*` crates transitively; upstream Solana stack has not migrated to bincode 2.x. No local fix.
  - `proc-macro-error 1.0.4` (RUSTSEC-2024-0370): pulled by `teloxide → aquamarine` for build-time macro doc rendering. Build-time only, never shipped in the binary.
  - `rustls-pemfile 1.0.4` (RUSTSEC-2025-0134): pulled by `teloxide-core → reqwest 0.11.27`. Fix path requires `reqwest 0.12` which needs a teloxide major bump (out of scope for Phase 11.1).
- **Fix:** Added all three to `.cargo/audit.toml` with D-13/D-14 justification and per-advisory expiry conditions.
- **Commit:** `ea4bfaf`.

### Reproduction Commands (verbatim from Plan 04 Task 3)

```bash
# 1. Environment
export DATABASE_URL="postgres://<dev-user>:<dev-pw>@localhost/tick_liq_dev"
# Any RPC_URL / KEYPAIR_PATH per project CLAUDE.md

# 2. Run watch for 5 minutes
cargo run --release -- watch \
    --pool <TARGET_POOL_ADDR> \
    --position-mint <POSITION_MINT> \
    --cex-symbol SOLUSDT 2>&1 | tee /tmp/phase11_1_smoke.log &
WATCH_PID=$!

sleep 300  # 5 minutes

kill -TERM $WATCH_PID
wait $WATCH_PID 2>/dev/null

# 3. Verify DB writes
psql "$DATABASE_URL" -c "SELECT price, recorded_at FROM pnl_history WHERE recorded_at > NOW() - INTERVAL '6 minutes' ORDER BY recorded_at DESC LIMIT 20;"

# 4. Verify log signals
grep -c 'cex_ws: connected, subscribed to bookTicker' /tmp/phase11_1_smoke.log  # expect >= 1
grep -c 'cex_ws: shutdown received' /tmp/phase11_1_smoke.log                    # expect 1
grep -c 'cex_ws: connect error\|disconnect error' /tmp/phase11_1_smoke.log      # expect 0

# 5. Price plausibility vs Binance SOLUSDT mid ± 1% (cross-check with https://www.binance.com/en/trade/SOL_USDT)
```

### Why this matters

binance-sdk v45 was wired in Plan 02 and exercised only by the adapter-level unit test (`apply_book_ticker_*`). The full async path (`SpotWsStreams::production().connect().await` → `book_ticker().await` → `on_message()` callback → `CexPriceState` writes → downstream `pnl_history` inserts) is not unit-covered. The 5-minute live smoke is the sole end-to-end verification that real Binance frames reach the database under production conditions. Until it runs, MIGRATE-04 (binance-sdk v45 wiring) is technically complete at the code-review level but un-proven in operation.
