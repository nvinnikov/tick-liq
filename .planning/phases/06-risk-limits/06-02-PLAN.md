**2. Add to `src/strategy/risk_monitor.rs` -- two new `impl RiskMonitor` methods:**

**`pub async fn load_or_init(pool: &PgPool, pool_address: &str) -> anyhow::Result<RiskState>`:**
- SELECT from risk_state WHERE pool_address = $1 using `sqlx_core::query::query` with `.fetch_optional(pool)`.
- If row exists: deserialize into RiskState. If `halt_flag = true`, emit `tracing::error!("risk: halt_flag set from previous session -- rebalancing will remain halted until DB is manually cleared")`.
- If no row exists: INSERT a fresh row with `peak_pnl = 0.0`, `pause_flag = false`, `halt_flag = false`, `current_drawdown_pct = 0.0`, `updated_at = NOW()`. Return the fresh RiskState.
- CRITICAL (per Pitfall 2 in RESEARCH.md): NEVER use `ON CONFLICT DO UPDATE SET halt_flag = false` in the init path. The SELECT + conditional INSERT pattern prevents overwriting an existing halt_flag.
- Use `sqlx_core::query::query` (non-macro path) to match the project pattern in writer.rs.
- Use `sqlx_core::row::Row` trait with `.get::<T, _>("column_name")` for column access.

**`pub fn persist_state(pool: PgPool, state: RiskState)`:**
- Fire-and-forget via `tokio::spawn`, matching `spawn_pnl_write` pattern exactly.
- Inside the spawned task: execute the upsert SQL:
  ```sql
  INSERT INTO risk_state (pool_address, peak_pnl, current_drawdown_pct, pause_flag, halt_flag, updated_at)
  VALUES ($1, $2, $3, $4, $5, NOW())
  ON CONFLICT (pool_address) DO UPDATE SET
    peak_pnl = EXCLUDED.peak_pnl,
    current_drawdown_pct = EXCLUDED.current_drawdown_pct,

    halt_flag = EXCLUDED.halt_flag,

  ```

// Drift User PDA: seeds = ["user", authority_pubkey, subaccount_0_le_bytes]
// Program ID: dRiftyHA39MWEi3m9aunc5MzRF1JYuBsbn6VPcn33UH
pub fn derive_drift_user_pda(authority: &solana_sdk::pubkey::Pubkey) -> solana_sdk::pubkey::Pubkey {
    let drift_program_id = solana_sdk::pubkey!("dRiftyHA39MWEi3m9aunc5MzRF1JYuBsbn6VPcn33UH");
    let (pda, _) = solana_sdk::pubkey::Pubkey::find_program_address(
        &[b"user", authority.as_ref(), &0u16.to_le_bytes()],
        &drift_program_id,
    );
    pda
}

2. Implement `pub fn fetch_drift_margin_ratio(&self) -> Option<f64>`:
